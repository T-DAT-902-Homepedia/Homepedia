
# Ubiquitous Language

## Qu'est-ce qu'un Ubiquitous Language ?

Un ubiquitous language (langage omniprésent) est un vocabulaire commun, partagé et compris par tous les membres d'une équipe projet, qu'ils soient techniques (développeurs) ou métiers (experts du domaine). Il est au cœur de la démarche Domain-Driven Design (DDD).

### Objectifs
- Réduire les malentendus entre les parties prenantes.
- Faciliter la communication et la collaboration.
- Aligner le code, la documentation et les échanges sur le métier.

### Comment le constituer ?
1. Collaborer régulièrement entre développeurs et experts métier.
2. Identifier et définir les concepts, termes et règles du domaine.
3. Documenter ces notions de façon claire et non ambiguë.
4. Utiliser ce vocabulaire dans tout le projet (code, docs, discussions).
5. Mettre à jour le langage au fil de l’évolution du projet.

Ce langage devient la référence pour tout le monde, assurant une compréhension partagée et une conception logicielle fidèle au métier.

## Notions du domaine immobilier

### Code INSEE

Le code INSEE est un identifiant numérique attribué par l'Institut National de la Statistique et des Études Économiques (INSEE) à chaque commune française. Il est utilisé pour référencer de manière unique une commune dans les bases de données administratives et immobilières. Ce code est essentiel pour l'identification précise des localisations dans les applications immobilières.
